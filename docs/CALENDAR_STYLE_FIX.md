**状态：** ✅ 已修复

## 🔍 问题描述

模拟盘和实盘的收益日历热力图样式不正确：

### 问题 1：数据显示错误
- **现象：** 收益率显示为 `0.01%` 而非 `1.00%`
- **原因：** 后端返回的 `daily_return` 是小数形式（0.0123 = 1.23%），前端未转换为百分比

### 问题 2：格子太小
- **现象：** 日历格子显示拥挤，收益率数字看不清
- **原因：** `min-height: 50px` 和 `padding: 2px` 太小

### 问题 3：颜色对比度不足
- **现象：** 涨跌颜色区分不明显
- **原因：** 使用纯色背景，缺乏层次感

---

## ✅ 修复方案

### 1️⃣ 数据格式修复

**文件：** `frontend/src/views/SimAnalysis.vue` 和 `RealAnalysis.vue`

```javascript
// 修复前
return: dayData ? dayData.return : 0

// 修复后
return: dayData ? (dayData.return * 100) : 0  // 转换为百分比
```

### 2️⃣ 样式优化

```css
/* 增大格子尺寸 */
.calendar-day {
  min-height: 60px;  /* 50px → 60px */
  padding: 4px;      /* 2px → 4px */
  justify-content: space-between;  /* 分散对齐 */
}

/* 增大字体 */
.day-num {
  font-size: 14px;  /* 13px → 14px */
}

/* 添加渐变背景和阴影 */
.calendar-day.gain-5 {
  background: linear-gradient(135deg, #dc3545 0%, #c82333 100%);
  box-shadow: 0 2px 6px rgba(220, 53, 69, 0.4);
}
```

## ⚠️ 重要说明：实盘 vs 模拟盘数据格式

**实盘和模拟盘的后端数据格式不同！**

| 表名 | 字段 | 格式 | 示例 | 前端处理 |
|------|------|------|------|----------|
| `daily_snapshots` (实盘) | `daily_return` | **百分比** | `1.8969` = 1.8969% | 直接使用 |
| `sim_daily_snapshots` (模拟盘) | `daily_return` | **小数** | `0.0123` = 1.23% | 乘以 100 |

**原因：** 两个表由不同系统写入，格式约定不一致。

**前端代码差异：**

```javascript
// RealAnalysis.vue (实盘) - 直接使用
return: dayData ? dayData.return : 0

// SimAnalysis.vue (模拟盘) - 转换为百分比
return: dayData ? (dayData.return * 100) : 0
```

## 🔧 追加修复：颜色分类逻辑

**修复时间：** 2026-04-11 05:31

### 问题

**现象：** +0.01% 和 -0.01% 显示相同颜色（绿色）❌

**原因：** `getReturnClass` 函数判断顺序错误

```javascript
// 错误逻辑
if (ret >= 1) return 'gain-1'
if (ret >= -1) return 'loss-1'  // ❌ 这里捕获了 -1 到 +∞ 的所有值！
```

当 `ret = 0.5` 时：
- `ret >= 1` → false
- `ret >= -1` → true → 返回 `loss-1`（绿色）❌

### 修复

```javascript
// 正确逻辑
if (ret === 0 || ret === undefined) return ''  // 无数据或 0%
if (ret >= 5) return 'gain-5'   // ≥+5% 深红
if (ret >= 3) return 'gain-3'   // +3%~+5% 中红
if (ret > 0) return 'gain-1'    // +0.01%~+3% 浅红 ✅
if (ret >= -1) return 'loss-1'  // -1%~0% 浅绿 ✅
if (ret >= -3) return 'loss-3'  // -3%~-1% 中绿
return 'loss-5'                 // ≤-3% 深绿
```

### 修复后效果

| 收益率 | 修复前 | 修复后 |
|--------|--------|--------|
| +2.5% | ❌ 绿色 | ✅ 浅红 |
| +0.5% | ❌ 绿色 | ✅ 浅红 |
| +0.01% | ❌ 绿色 | ✅ 浅红 |
| 0% | 灰色 | 灰色 ✅ |
| -0.01% | 浅绿 | 浅绿 ✅ |
| -0.5% | 浅绿 | 浅绿 ✅ |
| -2.5% | 浅绿 | 浅绿 ✅ |

## 🔧 追加修复：日历和月度汇总数据格式

**修复时间：** 2026-04-11 05:48

### 问题

**现象：** 
- 日历热力图显示异常大的值（如 189.69%）
- 月度汇总表格数据不正确

**原因：** 
1. 后端已返回百分比形式（`daily_return = 1.8969` 表示 1.8969%）
2. 但前端 `getCalendarDays` 又乘以 100，导致重复计算
3. 结果：`1.8969 * 100 = 189.69%` ❌

### 修复

**文件：** `frontend/src/views/SimAnalysis.vue`

```javascript
// 修复前
return: dayData ? (dayData.return * 100) : 0  // ❌ 重复乘以 100

// 修复后
return: dayData ? dayData.return : 0  // ✅ 直接使用百分比数据
```

### 数据流

```
数据库 (sim_daily_snapshots)
  ↓ daily_return = 1.8969 (百分比形式)
后端 API
  ↓ 返回 { return: 1.8969 }
前端 getCalendarDays
  ↓ 直接使用 return: 1.8969
日历热力图显示
  → 显示 1.89% ✅
```

### 月度汇总计算

后端月度汇总计算逻辑（正确）：

```python
# daily_return 是百分比形式（如 1.8969）
compounded = 1.0
for r in returns:
    compounded *= (1 + r / 100)  # 除以 100 转小数
monthly_ret = (compounded - 1) * 100  # 转回百分比
```

示例：
- 日收益率：1.5%, 2.0%, -0.5%
- 复利计算：`(1 + 0.015) * (1 + 0.02) * (1 - 0.005) - 1 = 0.03017`
- 月收益率：`3.017%`
